## 七、适用场景

### 什么时候选谁？

| 场景 | 推荐 | 原因 |
|------|------|------|
| **Next.js 全栈** | ✅ **Neon** | 免费额度大 + 分支强 |
| **需要内置 Auth** | ✅ **Supabase** | 开箱即用 |
| **想用 MySQL** | ✅ **PlanetScale** | Vitess 血统 |
| **小型 MVP** | ✅ **Neon** | 便宜够用 |
| **需要 Realtime** | ✅ **Supabase** | WebSocket 内置 |
| **电商/订单系统** | ✅ **PlanetScale** | MySQL 事务更强 |
| **AI 应用/RAG** | ✅ **Neon** | + Pinecone 向量 |

## 十、最终建议

### 针对你的 AI 出海站点

**推荐：Neon PostgreSQL**

```
优点：
├── 3GB 免费（够用）
├── 无限分支（开发友好）
├── PostgreSQL（功能全）
├── Prisma 原生支持
└── 价格便宜

组合：
Neon (数据库) + Prisma (ORM) + Supabase Auth (可选)
```

**如果你想要一站式解决方案：**

```
Supabase = 数据库 + Auth + Storage + Realtime
适合：不想自己组合多个服务
```
